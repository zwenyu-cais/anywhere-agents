# code-release

Pre-release audit checklist for research code repositories. Walks 10 categories of release-prep rules — secrets and `.env.template` parity, `environment.yml` reproducibility, README state (GPU spec, artifact location), licensing and vendored-fork attribution, conda/SLURM idioms, sweep-output `.gitignore` patterns, and repository hygiene around rename commits — distilled from real release-prep incidents.
